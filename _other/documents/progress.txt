            Intermediate steps:

            1) Get "ibm-industry" by mapping EU sector (table: organization.nace_section) to IBM nomenclature:
                EU Sector                   Mapped IBM Industry
                A – Agriculture             Industrial

                M – Professional Services   Services
                N – Administrative Services Services
                O – Public Administration   Public
                P – Education               Education
                Q – Healthcare              Healthcare
                R – Entertainment           Entertainment
                S – Other Services          Services

            2) "Daily Revenue" = Annual Revenue (table: organization.revenue_range) / Business days per year (table: organization.business_days_per_year)
                Annual Revenue is in a range. 
                    < euro 2M
                    2-10
                    20-50
                    50-250
                    250-1bil
                    > 1bil

                Take the midpoint in the range. For > 1bil, take 1bil

            3) "IT dependency level" = SUM(organization.manual_operation, organization.production_dependency, organization.customer_access)

            4) IT factor = 
                if "IT dependency level" <= 1 -> 0.30 (Low)
                if "IT dependency level" <= 3 -> 0.60 (Medium)
                if "IT dependency level" > 3 -> 0.90 (High)

            5) "Sector Factor" = 
                (IBM Industry   -> value)
                Healthcare      ->  1.1
                Financial       ->  1.01842105263158
                Industrial      ->  0.993859649122807
                Energy          ->  0.98640350877193
                Technology      ->  0.984649122807018
                Pharmaceuticals ->  0.976754385964912
                Services        ->  0.974561403508772
                Entertainment   ->  0.968859649122807
                Media           ->  0.959649122807018
                Hospitality     ->  0.951315789473684
                Transportation  ->  0.949122807017544
                Education       ->  0.941228070175439
                Research        ->  0.940789473684211
                Communications  ->  0.939035087719298
                Consumer        ->  0.937719298245614
                Retail          ->  0.929824561403509
                Public          ->  0.9

            6) "Adjusted Daily Loss" = "Daily revenue" * "IT factor" * "sector factor"

            7) "Downtime GAP score" = 
                    ((AVERAGE(
                        Daily Backups
                        Offsite / Immutable Backups
                        Restore Testing
                        Incident Response Plan
                        Defined Roles
                        Incident Exercises
                        Infrastructure Resilience
                        Monitoring
                    ) - 1)/4)*100

                    These maps to:
                    ┌─────────────────────────────┬────────────────────────────────────────────────────────────────────────────────────────┐
                    │            Item             │                                     CIS Safeguard                                      │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Daily Backups               │ 11.2 — Perform Automated Backups                                                       │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Offsite / Immutable Backups │ 11.4 — Establish and Maintain an Isolated Instance of Recovery Data                    │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Restore Testing             │ 11.5 — Test Data Recovery                                                              │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Incident Response Plan      │ 17.2 — Establish and Maintain Incident Response Process                                │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Defined Roles               │ 17.1 — Designate Personnel to Manage Incident Handling                                 │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Incident Exercises          │ 17.5 — Test Incident Response                                                          │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Infrastructure Resilience   │ 11.1 — Establish and Maintain a Data Recovery Process (closest match under Control 11) │
                    ├─────────────────────────────┼────────────────────────────────────────────────────────────────────────────────────────┤
                    │ Monitoring                  │ 8.11 — Conduct Audit Log Reviews (or 8.2 Collect Audit Logs, depending on scope)       │
                    └─────────────────────────────┴────────────────────────────────────────────────────────────────────────────────────────┘

            8) "Downtime Days" =
                if "Downtime GAP score" = 80 => 0.5 (High resilience)
                if "Downtime GAP score" = 65 => 1.0 (Medium resilience)
                if "Downtime GAP score" = 45 => 2.0 (Low resilience)
                if "Downtime GAP score" = 9 => 3.0 (Very low resilience)

            9) Finally =>
                "Downtime cost (mid)" = "Adjusted Daily Loss" * "Downtime Days"
                "Downtime cost (low)" = "Downtime cost (mid)" * 0.7
                "Downtime cost (high)" = "Downtime cost (high)" * 1.3

            10) If any value in the intermediate step is missing, list missing values, instead.

            11) Summary / Calculation logic:
                  1. NACE section → IBM industry → sector factor
                  2. Revenue range → annual revenue midpoint ÷ business days = daily revenue
                  3. manualOperation + productionDependency + customerAccess → IT dependency level → IT factor (0.30 / 0.60 / 0.90)
                  4. Adjusted daily loss = daily revenue × IT factor × sector factor
                  5. Downtime GAP score = ((avg currentCmmi of 8 safeguards − 1) / 4) × 100 → downtime days (0.5 / 1.0 / 2.0 / 3.0)
                  6. Mid = adjusted daily loss × downtime days; Low = mid × 0.7; High = mid × 1.3
